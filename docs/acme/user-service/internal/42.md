# User Service — Internal Reference

## Overview

The `UserService` class provides multi-tenant user management scoped strictly to an organisation. All methods enforce the `orgId` boundary — it is not possible to retrieve or mutate users from another org through this interface.

## Methods

### `getUser(userId: string): Promise<User | null>`

Retrieves a single user by internal ID. Returns `null` if the user does not exist or does not belong to this service's org.

**Throws** if the database query fails.

### `deactivate(userId: string, reason: string): Promise<void>`

Deactivates a user account. Only callable by admins. Records the deactivation reason in the audit log.

**Permission required:** `ADMIN` role.

## Key decisions

1.  **No cross-org queries.** Every method on this class is bound to `this.orgId` at construction time. Passing a `userId` from another org returns `null`.
    
2.  **Soft deactivation only.** Users are never hard-deleted — this preserves audit history. Deactivated users cannot log in but their records are retained.
